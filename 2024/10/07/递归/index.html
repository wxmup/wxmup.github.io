递归的定义

是解决方法的一种.

递归将问题不断分解为小问题, 直到子问题可以用普通的方法解决.

认识递归

我们从简单的问题开始学习递归.

计算一列数之和

给定一堆数字, 计算出这些数字的总和.

例如，我们要计算下列数字之和：

1 + 2 + 3 + 4 + 5

我们可以把这个问题看成：
sum = 1 + （2 + 3 + 4 + 5） = 1 + a
a = 2 + (3 + 4 + 5) = 2 + b
b = 3 + (4 + 5) = 3 + c
c = 4 + 5 = 4 + d
d = 5

这样，我们发现了这个问题，总是可以看成是这个样子的：

  • 前五个数字之和 = 第一个数字 + 后四个数字之和
  • 后四个数字之和 = 第一个数字 + 后三个数字之和
  • …

我们还发现了，停止分解这个问题的条件是：
数字列表长度为 1

递归三原则

  1. 递归必须有停止递归的条件

    • 当问题不能再分解了，应该停止递归

  2. 要让问题不断分解, 使其逐渐逼近停止递归的条件

  3. 递归算法要不断地调用自己

!汉诺塔问题

汉诺塔问题由法国数学家爱德华·卢卡斯于 1883 年提出。

有 3 根柱子和 64 个依次叠好的金盘子，下面的盘子大，上面的盘子小。你的任务是将 64 个叠好的盘子从一根柱子移动到另一根柱子.
同时有三个重要的限制条件：每次只能移动一个盘子，并且大盘子不能放在小盘子之上, 并且只能从上到下的顺序取盘子.
你夜以继日地移动盘子（每一秒移动一个盘子），试图完成任务。

思路和解决方法

假设有一个四层的汉诺塔的盘子在起始柱a上,有一个辅助柱b,要将a柱上的盘子移动到c柱上.
现在,有四层汉诺塔的盘子在a柱,要移动到c柱上,需要做三件事:

  1. 把前三层(前三层的顺序是指自上而下的顺序)的盘子移动到b柱
  2. 然后把最后一个盘子(同样按照是自上而下的顺序)移动到c柱
  3. 最后,我要把b柱上的三层汉诺塔的盘子移动到c柱上.

那么,如何完成第一件事呢?即如何把前三层的盘子移动到b柱上呢?事实上考虑前三层的汉诺塔如何移动到b柱上的这个问题,等价于考虑a柱上的一个三层汉诺塔的盘子到b柱上的问题.并且,移动到b柱和移动到c柱使用的方法上一样的.为了让三层汉诺塔的盘子移动到b柱,需要完成三件事.

  1. 前两层汉诺塔的盘子移动到c柱.
  2. 将最后一层的盘子移动到b柱.
  3. 将刚刚放置到c柱上的前两层汉诺塔的盘子移动到b柱

为了完成上一个步骤的第一件事,让前两层汉诺塔的盘子移动到c柱,也就是让a柱上只有两层的汉诺塔的盘子移动到c柱上.仍然需要完成三件事:

  1. 前一层的汉诺塔的盘子移动到b柱
  2. 将最后一个盘子移动到c柱
  3. 将刚刚放置到b柱上的一层汉诺塔的盘子移动到c柱
    这一次可以同时完成三件事.第一件事可以直接完成,因为最后一个盘子可以直接移动到到b柱上.这就是说,让前两层汉诺塔的盘子移动到c柱,这件事已完成.让我们回到上一个步骤,即把前三层的盘子移动到b柱上.

对于把前三层的盘子移动到b柱上这个问题来说,第一个事情已完成,第二个事情也可以直接完成.对于第三个事情来说,如何将刚刚放置到c柱上的前两层汉诺塔的盘子移动到b柱,事实上等价于,以c柱为起始点的两层汉诺塔,如何移动到b柱上.这个分解方法和移动步骤和之前如出一辙.具体来说,仍然上三个步骤:

  1. 要把第一层的盘子移动到c柱上
  2. 移动最后一个盘子到b柱
  3. 最后将刚刚放置的第一层的盘子移动到b柱上.
    也就是说,解决第三件事的方法,除了起始柱和目标柱不一样,整体方法完全一样.
    这样,第三件事就解决了.我们再回到上一级步骤,即将在a柱上的四层汉诺塔的盘子移动到c柱上.

对于在a柱上的四层汉诺塔的盘子移动到c柱上来说,第一件事情已经解决,第二件事情也可以直接解决.第三件事的解决步骤和解决四层汉诺塔的盘子移动到c柱上的方法来说上一样的.

因此,我们知道了,如果要递归解决这个问题,要设计一个具备如下功能的函数:
将总层数为x的的汉诺塔盘子,从起始柱a移动到目标柱c,这个过程借助了辅助柱b.
这个函数应该具备这些参数,function(x层数,起始柱a,辅助柱b,目标柱c).

具体来说,为了实现这个函数的功能,需要:

  • 将总层数x-1的汉诺塔的盘子从起始柱移动到辅助柱
  • 将总层数x的汉诺塔的最后一个盘子从起始柱移动移动到目标柱
  • 将总层数x-1的汉诺塔的盘子从辅助柱移动到目标柱

刚才的问题’如何将前三层的盘子移动到b柱上这个问题’的解决方放里的要解决的第三件事情表明,起始柱和目标柱需要发生调换.

至此,这个问题已解决.
